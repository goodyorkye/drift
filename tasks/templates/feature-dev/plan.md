# 功能开发 - 方案阶段

## 目标

分析需求，产出详细的实现方案。本阶段**只读不写**，产出方案文档后等待人工确认。

## 执行流程

1. 读取任务参数中的 `description`、`goal`、`acceptance`
2. 读取 `targetRepo` 相关代码，理解现有架构
3. 制定实现方案，包含：
   - 需要新增/修改的文件列表
   - 关键实现决策及理由
   - 潜在风险和依赖
4. 保存方案到 `reports/{YYYY-MM-DD}/plan-{taskId}.md`

## 方案文档格式

```markdown
# 实现方案：{任务标题}

## 需求理解
（复述目标和验收标准，确认理解一致）

## 实现思路

## 文件变更清单
| 文件 | 操作 | 说明 |
|------|------|------|

## 关键决策
（技术选型、设计取舍）

## 风险与注意事项

## 实现顺序
1. ...
2. ...
```

## 成功标准

- 方案文档完整，包含文件变更清单和实现顺序
- 写入 result.json `{"status": "success", "outputFile": "reports/..."}`
- 等待人工通过 `drift task approve <id>` 确认后，进入实现阶段
