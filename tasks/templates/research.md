# 调研任务模板

## 最高优先级原则

只读取信息，只产出报告。禁止修改任何外部资源。

## 执行流程

1. 读取任务参数（见下方注入的 JSON）
2. 根据 `description`、`goal`、`constraints` 确定调研范围
3. 使用 WebSearch / WebFetch 收集信息，至少 3 个独立来源
4. 整理关键发现，撰写报告
5. 保存报告到 `reports/{YYYY-MM-DD}/research-{关键词}.md`
6. 将结果写入 Result Contract 文件

## 决策规则

- 遇到付费墙或访问限制：跳过该来源，使用替代资源
- 信息不足：基于已有信息做最佳推断，在报告中标注假设
- 来源冲突：注明冲突，列出各方观点

## 报告格式

```markdown
# {任务标题}

日期：YYYY-MM-DD

## 概述
（一句话总结核心发现）

## 核心发现
- 发现 1
- 发现 2

## 详细内容
（按 constraints 定义的边界展开）

## 结论与建议
（对应 goal 的具体结论）

## 参考链接
- [来源 1](url)
- [来源 2](url)
```

## 成功标准

- 报告包含概述、核心发现、参考链接
- 结论明确回应了任务的 `goal` 和 `acceptance` 字段
- 报告以中文撰写，技术术语可保留英文原文
