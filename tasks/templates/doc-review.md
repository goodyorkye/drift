# 文档审查模板

## 最高优先级原则

禁止修改目标目录任何文件。只读取文档，只产出审查报告。

## 执行流程

1. 扫描目标目录，获取所有文档文件列表（优先 .md，含其他文档格式）
2. 逐文件检查：
   - **正确性**：事实错误、技术错误、逻辑矛盾
   - **一致性**：与其他文档的矛盾
   - **格式**：markdown 语法、链接有效性、标题层级
   - **优化空间**：内容冗余、结构不合理
3. 跨文档分析：重叠内容、可合并文档、整体洞察
4. 保存报告到 `reports/{YYYY-MM-DD}/doc-review-{目录名}.md`

## 报告格式

```markdown
# 文档审查报告：{targetRepo 或目标目录}

日期：YYYY-MM-DD
范围：{文件数量}个文档

## 概述

## 发现的问题

### 错误类（需修复）
| 问题 | 文件 | 行号 | 说明 |

### 矛盾类（需澄清）
| 文档 A | 文档 B | 矛盾内容 |

### 格式类（建议修复）
| 问题 | 文件 | 说明 |

### 可合并文档
| 文档列表 | 建议 |

## 洞察发现
- 新研究方向：
- 项目缺陷：
- 重大未解决问题：

## 建议操作清单
```

## 成功标准

- 至少发现 3 个有价值的问题或洞察（或确认文档质量良好并说明理由）
- 问题有具体文件名和位置
- 报告以中文撰写
