# 调研任务创建指南

这个 guide 用于帮助创建 `research` 类型任务的 `task.md`。它是补充材料，不是强制模板。

## 任务边界

调研任务适合：

- 收集和核对公开信息
- 对比多个方案、产品、技术路线或市场事实
- 形成结构化中文报告
- 给出有来源支撑的结论、风险和建议

调研任务不适合：

- 修改代码、配置或外部系统
- 代表用户做购买、注册、下单等操作
- 在没有来源依据时编造事实

## 创建时建议澄清的信息

- 调研主题是什么
- 用户真正要回答的问题是什么
- 期望覆盖哪些范围，哪些范围明确不看
- 是否需要最新信息
- 是否需要引用链接、发布时间、版本号、价格、法律条款等精确信息
- 是否偏向事实梳理、方案比较、风险评估、竞品分析或行动建议
- 报告读者是谁，例如个人决策、团队评审、管理层汇报
- 输出语言、深度和篇幅偏好

## task.md 推荐内容

可以自由组织，但建议包含：

- 背景：为什么要做这次调研
- 目标：最终需要回答的问题
- 范围：必须覆盖和明确排除的内容
- 约束：时间、地区、行业、技术栈、预算、来源要求等
- 产出：希望得到报告、表格、结论清单还是建议清单
- 成功标准：什么样的结果算完成

## 执行建议

执行 agent 通常应：

- 优先读取 `task.md` 理解调研目标
- 使用多个独立来源互相校验
- 对时间敏感信息标明查询日期或来源日期
- 遇到来源冲突时说明冲突，而不是强行合并
- 信息不足时明确写出假设和不确定性
- 不修改外部资源，只产出调研结果

## 报告结构建议

可参考以下结构，不强制：

```markdown
# 调研报告：{主题}

## 概述

## 关键结论

## 背景与范围

## 详细发现

## 风险与不确定性

## 建议

## 参考来源
```

## 不要继承的旧约定

- 不要求任务输入是固定 JSON 字段
- 不要求固定保存到 `reports/` 目录
- 不要求在 `task.md` 中重复系统结果协议
